# CDN 功能重构方案

**创建时间**：2026-03-11 13:23
**状态**：待审核
**优先级**：中

## 执行摘要

本方案旨在重构现有 CDN 功能，支持两种 CDN 模式：
1. **回源型 CDN**（现有功能，保留）：适用于 Cloudflare、阿里云 CDN 等支持自动回源的服务
2. **对象存储型 CDN**（新增功能）：适用于 DogeCloud、七牛云、又拍云等需要上传文件的服务

## 目标与收益

### 核心目标
- 保留现有回源型 CDN 功能，确保向后兼容
- 新增对象存储型 CDN 支持，自动同步静态资源到 DogeCloud
- 提供统一的管理界面，支持两种模式的配置和切换
- 实现自动化部署流程，减少手动操作

### 预期收益
- **灵活性提升**：支持更多 CDN 服务商选择
- **性能优化**：静态资源通过 CDN 加速，提升全球访问速度
- **运维简化**：自动同步机制，无需手动上传文件
- **成本优化**：可根据流量和预算选择合适的 CDN 方案

## 现状分析

### 现有 CDN 实现

**数据库结构**：
- 表：`site_settings`
- 字段：`cdn_asset_url`（varchar 255，nullable）

**配置流程**：
1. 管理员在"站点设置"页面填写 CDN 域名
2. 系统将 `cdn_asset_url` 写入 `config('app.asset_url')`
3. Laravel 自动将所有 `asset()` 和 `mix()` 生成的 URL 替换为 CDN 域名

**适用场景**：
- Cloudflare（免费，支持回源）
- 阿里云 CDN / 腾讯云 CDN（按流量计费，支持回源）
- 任何支持"源站配置"的 CDN 服务

**局限性**：
- 不支持需要手动上传文件的对象存储服务
- 无法使用 DogeCloud、七牛云等对象存储 + CDN 方案
- 缺少文件同步和版本管理机制

### 核心组件设计

#### 1. CDN 管理器（CdnManager）

**职责**：
- 统一管理两种 CDN 模式
- 提供模式切换和配置验证
- 协调存储服务和同步任务

**接口设计**：

```php

interface CdnManager
{
    public function getMode(): CdnMode;
    public function setMode(CdnMode $mode): void;
    public function getAssetUrl(string $path): string;
    public function isConfigured(): bool;
    public function validateConfiguration(): array;
}
```

#### 2. 存储服务抽象层（StorageProvider）

**职责**：
- 抽象不同存储服务商的 API 差异
- 提供统一的文件上传、删除、列表接口
- 处理认证和签名逻辑

**接口设计**：

```php
namespace App\Support\Cdn\Storage;

interface StorageProvider
{
    public function upload(string $localPath, string $remotePath): bool;
    public function delete(string $remotePath): bool;
    public function exists(string $remotePath): bool;
    public function listFiles(string $prefix = ''): array;
    public function getPublicUrl(string $remotePath): string;
    public function validateCredentials(): bool;
}
```

**实现类**：
- `DogeCloudStorageProvider`
- `QiniuStorageProvider`
- `UpyunStorageProvider`

#### 3. 文件同步服务（CdnSyncService）

**职责**：
- 扫描本地静态资源目录
- 对比本地和远程文件差异
- 批量上传变更文件
- 记录同步日志和状态

**核心方法**：

```php
namespace App\Support\Cdn;

class CdnSyncService
{
    public function syncAll(): SyncResult;
    public function syncDirectory(string $directory): SyncResult;
    public function syncFile(string $filePath): bool;
    public function getDiff(): array;
    public function clearRemote(): bool;
}
```

#### 4. DogeCloud 适配器（DogeCloudStorageProvider）

**认证方式**（基于通用对象存储实践）：

```php
class DogeCloudStorageProvider implements StorageProvider
{
    private function generateSignature(string $method, string $path, array $params = []): string
    {
        // 1. 构建待签名字符串
        $stringToSign = implode("\n", [
            $method,
            $path,
            http_build_query($params),
            $this->accessKey,
        ]);

        // 2. HMAC-SHA256 签名
        $signature = hash_hmac('sha256', $stringToSign, $this->secretKey);

        return $signature;
    }

    public function upload(string $localPath, string $remotePath): bool
    {
        $url = $this->endpoint . '/' . $this->bucket . '/' . ltrim($remotePath, '/');

        $signature = $this->generateSignature('PUT', $remotePath);

        $response = Http::withHeaders([
            'Authorization' => "DOGE {$this->accessKey}:{$signature}",
            'Content-Type' => mime_content_type($localPath),
        ])->attach('file', file_get_contents($localPath), basename($remotePath))
          ->put($url);

        return $response->successful();
    }
}
```

**注**：由于无法访问 DogeCloud 官方 API 文档，上述实现基于通用对象存储 API 模式（类似 AWS S3、阿里云 OSS）。实际实施时需要参考 DogeCloud 官方文档调整。

## 使用场景示例

### 场景 1：使用 Cloudflare（回源型 CDN）

**配置步骤**：
1. 在 Cloudflare 添加站点，配置 DNS
2. 在管理后台选择"回源型 CDN"
3. 填写 CDN 域名：`https://cdn.example.com`
4. 保存配置

**效果**：
- 所有静态资源 URL 自动替换为 `https://cdn.example.com/build/assets/...`
- Cloudflare 自动从源站拉取文件并缓存
- 无需手动上传或同步

### 场景 2：使用 DogeCloud（对象存储型 CDN）

**配置步骤**：
1. 在 DogeCloud 创建存储桶，获取 AccessKey 和 SecretKey
2. 配置 CDN 加速域名
3. 在管理后台选择"对象存储型 CDN"
4. 选择"DogeCloud"，填写凭证和存储桶信息
5. 启用"自动同步"和"构建时同步"
6. 点击"立即同步"进行首次全量同步

**效果**：
- 首次同步：上传所有静态资源到 DogeCloud
- 后续构建：自动检测变更文件并增量同步
- 所有静态资源 URL 替换为 DogeCloud CDN 域名
- 用户访问时直接从 DogeCloud CDN 获取文件

## 风险与缓解

### 风险 1：DogeCloud API 文档不完整

**影响**：可能导致 API 调用失败或认证错误

**缓解措施**：
1. 实施前联系 DogeCloud 技术支持，获取完整 API 文档
2. 参考其他用户的开源实现（如 1Panel SSL 证书推送）
3. 先实现通用接口，后续根据实际 API 调整
4. 提供详细的错误日志，便于调试

### 风险 2：大量文件同步耗时过长

**影响**：首次同步可能需要数分钟，影响用户体验

**缓解措施**：
1. 使用队列异步处理同步任务
2. 实现进度条和实时状态更新
3. 支持断点续传和失败重试
4. 提供"仅同步变更文件"选项

### 风险 3：密钥泄露风险

**影响**：AccessKey 和 SecretKey 泄露可能导致存储桶被恶意访问

**缓解措施**：
1. 使用 Laravel 加密存储敏感字段
2. 管理界面显示时脱敏（显示为 `****`）
3. 添加 IP 白名单限制（如果服务商支持）
4. 定期轮换密钥
5. 记录所有 API 调用日志

### 风险 4：向后兼容性

**影响**：现有使用回源型 CDN 的用户可能受影响

**缓解措施**：
1. 默认 CDN 模式为 `origin`，保持现有行为
2. 迁移脚本自动将现有 `cdn_asset_url` 映射到回源模式
3. 提供平滑升级路径，无需用户手动配置
4. 在 CHANGELOG 中明确说明变更内容

### 术语表

| 术语 | 说明 |
|------|------|
| 回源 | CDN 节点从源站自动拉取文件的机制 |
| 对象存储 | 云端文件存储服务，通常按存储量和流量计费 |
| AccessKey | 访问密钥，用于 API 认证 |
| SecretKey | 密钥，用于生成 API 签名 |
| 存储桶 | 对象存储中的文件容器，类似文件夹 |
| 增量同步 | 仅同步变更的文件，而非全量同步 |
